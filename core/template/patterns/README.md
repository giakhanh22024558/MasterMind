## How to add a new pattern

1. Create `<pattern-name>.md` in this folder
2. Follow the structure:
   - **Problem** — what concern motivates this pattern
   - **Solution** — the structural approach
   - **Trade-offs** — pros and cons
   - **Worked example** — concrete demonstration with placeholders
   - **When NOT to use** — boundary conditions
   - **Cross-references** — other patterns / meta-patterns related

If the pattern is an instantiation of a [meta-pattern](../../meta/), reference back to the meta-pattern instead of re-explaining its general lifecycle.

## When to promote a pattern to `meta/`

If a pattern is appearing in 3+ skills, it's no longer skill-specific. Promote it to [`meta/`](../../meta/) using the [defer-then-promote pattern](../../meta/defer-then-promote-pattern/).
